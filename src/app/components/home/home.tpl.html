CSS Components

Have you ever started a project saying "Not this time... this time the CSS will be very well structured" and a week after it became a big mess? ...

... it happened to me plenty of times

Thanks to CSS compilers like Sass or Less we can arrange the code in different files and folders.

Although it won't help us if we can't create a thoughtful structure for our code.

Design and Semantic: how to keep them together without messing each-other

In my experience these approaches lead us into issues:

  • Bind styles to tag selectors (chained or not)
    body section article h2 {...}
  • Using tags because how they look instead of what they mean
    <h1>Main Article title</h1> instead of <h2 class="main__title">Main Article title</h2>
  • Apply the same styles to different selectors over and over
    nav li a {cursor: pointer;} and some lines after aside li a {cursor: pointer;}

Design and Semantic: how to keep them together without messing each-other

How to feel you don't have to choose in between Design and Semantic, without writing yourself over and over:

  • Choose the markup for its meaning and not for how it looks by default
  • Create CSS components (if possible helped by some compiler) to standarize the team's code
  • Make those components with: variables, classes, mixins and some other ways to modularize code that might help you based on your experiences
  • Have fun creating them, is the best way to get more and more creative

Sass variables

As in programming languages, CSS compilers can use variables to enhance the code before rendering.

Sass variables

Project's colours

Despite how many of them or which, every project has colours.

Every brand also has very well defined a specific value for each color.

Caching those values helps the dev team to not think over and over about them (cleaner way of working).

Also we could use a map to store them all into a single variable.

Sass variables

Default font size

Standarize components, is not caching colours only. We can cache lot of different elements we use in the site-wide.

The project's font size is one of those value we have to make sure persist properly.

Sass variables

Img routing

One of those things give me headaches is to think where images are placed within the source code.

To caching the route not only helps to focus on what is important, also give us the power to change the route if is required in the future. A good example is migrating the images into a CDN outside the build and probably into a different domain.
